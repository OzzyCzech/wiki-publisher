# Starlight Action

GitHub Action that builds an [Astro Starlight](https://starlight.astro.build/) site from a Markdown repository and uploads it as a GitHub Pages artifact. Zero config required — drop in your `*.md` files and ship.

A bundled Starlight skeleton lives in [`template/`](./template). At minimum, the caller repo just needs some Markdown — everything else (config, components, assets, homepage) is optional and falls back to sensible template defaults.

**Features**

- Markdown + MDX content → static Starlight site, auto-deployed to GitHub Pages
- Sidebar autogenerated from your folders, or fully custom via `starlight.config.mjs`
- Auto-generated **Open Graph images** per page (Satori + Resvg, full Czech/Polish/Vietnamese coverage)
- Built-in **YouTube embed** remark plugin — bare `youtube.com` / `youtu.be` links auto-render as iframes
- Internal-link rewriting to relative URLs via the bundled rehype plugin
- Optional `.astro` components (`@components/*`, `@assets/*` aliases)
- Drop-in custom `public/` (favicon, OG image, CNAME, robots.txt)
- Override Starlight or Astro version per-project without forking
- Built-in raw-markdown endpoint at `/path.md` and `/rss.xml` feed
- Local preview with `npm run dev` inside `template/`

## Caller repo layout

```
<repo-root>/
├── starlight.config.mjs       # optional — site URL, sidebar, title, social, components map
├── src/                       # optional — overlays the bundled template/src
│   ├── components/            #   *.astro components referenced from starlight.config.mjs
│   ├── assets/                #   logo and other src-time assets
│   └── style.css              #   extra CSS injected via Starlight customCss
├── public/                    # optional — favicon, og.png, CNAME, robots.txt, ...
├── astro.config.mjs           # optional — full astro.config override (advanced)
├── index.md                   # optional — homepage (falls back to README.md, then to template default)
└── <content folders>/         # any markdown / mdx
```

Everything is optional. Without a `starlight.config.mjs` the site is built with `{ title: 'Site' }` and a sidebar autogenerated from your folders.

`src/`, `public/`, `starlight.config.mjs`, `astro.config.mjs` and standard repo files (`.git`, `.github`, `CLAUDE.md`, `README*`, `LICENSE`, `package*.json`, `tsconfig.json`, `node_modules`, `dist`, `.astro`) are excluded from the markdown content sync. Everything else is copied into `src/content/docs/`.

## `starlight.config.mjs`

All keys are optional.

```js
export default {
  site: 'https://example.com',          // optional — without it, RSS and sitemap are skipped
  linksHostname: 'example.com',         // optional — internal-link detection for the rehype links plugin
  rss: {                                // optional — drives /rss.xml when site is set
    title: "My notes",
    description: 'Personal notes',
  },
  starlight: {                          // any StarlightUserConfig option
    title: 'My notes',
    customCss: ['./src/style.css'],
    components: { Head: './src/components/Head.astro' },
    sidebar: [ /* ... */ ],
  },
  og: {                                 // optional — set `og: false` to disable
    brand: 'My notes',
    domain: 'example.com',
    tagline: 'Optional uppercase eyebrow',
    accent: ['#4f46e5', '#7c3aed'],     // gradient stops for accent bar / blooms
    bgGradient: 'linear-gradient(135deg, #fafafa 0%, #f1f5f9 100%)',
    logo: './public/logo.svg',          // optional, embedded as data URI
  },
};
```

Paths inside `starlight.config.mjs` are interpreted from the project root: `./src/components/Foo.astro`, `./src/assets/logo.png`, `./src/style.css`. The same paths work in your caller repo (since you mirror the `src/` layout) and in the build workdir.

You can also use the path aliases `@components/*` and `@assets/*` in MDX and component imports.

## Usage

```yaml
# .github/workflows/deploy.yml
name: Deploy
on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
      - uses: OzzyCzech/starlight@v2

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - id: deployment
        uses: actions/deploy-pages@v5
```

## Open Graph images

Every docs page gets an auto-generated 1200×630 PNG at `/og/<slug>.png`, and the action injects `og:image` / `twitter:image` / `twitter:card` meta tags into every page's `<head>`. Rendering is done with [Satori](https://github.com/vercel/satori) (HTML/CSS → SVG) and [Resvg](https://github.com/yisibl/resvg-js) (SVG → PNG); fonts are bundled (Inter Regular + Bold) so non-ASCII titles render correctly without external font fetches.

- Configure brand, domain, accent colors, gradient and logo via the `og` block in `starlight.config.mjs`.
- Per-page override: set `ogImage:` (absolute URL or site-relative path) in frontmatter — the action skips generating that image and uses your URL in meta tags.
- Disable entirely with `og: false`.
- **Custom layout**: drop your own `src/og/renderer.tsx` into the caller repo — it overlays the bundled default and gets called with `{ title, description, config }`.

## YouTube embeds

Plain links to YouTube auto-render as embedded iframes (`accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture` allowed). Both URL forms are supported:

```markdown
[Watch](https://www.youtube.com/watch?v=dQw4w9WgXcQ)
[Watch](https://youtu.be/dQw4w9WgXcQ)
```

## Inputs

| Name                | Default              | Description |
|---------------------|----------------------|-------------|
| `content-path`      | `.`                  | Where in the caller repo content lives |
| `node-version`      | `25`                 | Node version for `setup-node` |
| `base`              | _(none)_             | Astro `base` path (e.g. `/my-repo` for project pages) |
| `starlight-version` | _(pinned in template)_ | Override `@astrojs/starlight` version (e.g. `0.40.1`) |
| `astro-version`     | _(pinned in template)_ | Override `astro` version (e.g. `^6.3.0`) |

### Version overrides

The action ships locked dependency versions in `template/package-lock.json` for reproducible builds. Use `starlight-version` / `astro-version` only when you need to unblock a build or test a newer release without forking the action:

```yaml
- uses: OzzyCzech/starlight@v2
  with:
    starlight-version: '0.40.1'
```

When either override is set the action runs `npm install` (lockfile would disagree); otherwise it runs `npm ci` for determinism.

### Default title

If the caller has no `starlight.config.mjs`, the site title defaults to the GitHub repo name (`OzzyCzech/foo` → `foo`) instead of a generic `Site`.

## Local preview

The `template/` directory is a complete Astro project with a default `starlight.config.mjs` and a placeholder `index.md`, so it runs out of the box:

```bash
cd template
npm install
npm run dev
```

To preview a real caller's content locally, drop content into `template/src/content/docs/` and (optionally) replace `template/src/starlight.config.mjs` — both files are overwritten by the action at build time, so changes here are safe and won't leak into deployments.
